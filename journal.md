# [2단계] 문제 정의: 내 주변의 '가시' 찾기 (10분)

## 문제 정의
- 매일 운동량을 기록하고 싶지만, 오늘 얼마나 걸었는지, 뛴 시간과 칼로리 소모를 쉽게 확인하기 어렵다.
- 운동 일지를 쓰려면 매번 생각해서 기록해야 해서 번거롭고 꾸준히 못 하게 된다.
- 특히 공부와 일상 속에서 운동 습관을 만들려면 빠르고 직관적인 입력 화면이 필요하다.

## 주변 문제 사례
- 학교: "체육 시간에 뛴 시간과 칼로리를 바로 기록하고 싶어", "방과 후 운동량을 잊어버리면 측정하기 어려워"
- 일상: "오늘 몇 걸음 걸었는지 간편하게 확인하고 싶어", "헬스장에서 운동 세트와 시간을 간단히 입력하고 싶어"
- 심리: "오늘 운동했는지 기록하면 뿌듯함을 느끼고 싶어", "운동 습관을 유지하려면 작은 보상이 필요해"

## 활용할 수 있는 위젯 연결
- 입력창: 운동 종류, 운동 시간, 걸음 수, 거리 같은 숫자와 텍스트를 기록하는 데 사용
- 슬라이더: 오늘의 운동 강도나 만족도, 목표 달성 정도를 직관적으로 선택하도록 사용
- 버튼: 기록 저장, 초기화, 결과 보기와 같은 동작 트리거
- 체크박스/라디오: 운동 유형(걷기, 달리기, 스트레칭)이나 목표 달성 여부 선택
- 텍스트 출력: 오늘의 운동 결과 요약, 칼로리 소모 안내, 칭찬 메시지 표시

# [3단계] 기획: journal.md에 설계도 그리기 (20분)

## 1. 문제 정의: 왜 이 앱을 만드는가?
- 이 앱은 사용자가 매일 운동량을 쉽게 기록하고, 운동 습관을 유지하도록 돕기 위해 만든다.
- 복잡한 계산이나 여러 화면 이동 없이 한 번에 운동 정보(운동 종류, 시간, 강도, 걸음 수)를 입력하고 결과를 확인할 수 있어야 한다.
- 운동을 기록하면서 성취감과 동기 부여를 얻고, 꾸준히 운동량을 관리할 수 있도록 간결한 흐름을 제공한다.

## 2. 사용자 시나리오: 누가, 언제 사용하는가?
- 학생 A는 학교가 끝난 후 집에 와서 오늘 체육시간과 방과 후 산책 거리를 빠르게 기록한다.
- 직장인 B는 출근 전 또는 퇴근 후 짧은 운동을 마친 뒤 오늘의 운동 시간과 운동 강도를 입력한다.
- 운동 목표를 세우려는 사용자 C는 매일 저녁 잠들기 전에 하루 운동량을 정리하고 기록한다.

## 3. 필요한 위젯: 어떤 버튼과 입력창이 필요한가?
- 제목 표시: 앱 이름 또는 오늘의 운동 기록 안내문
- 텍스트 입력창: 운동 종류 입력 (예: 걷기, 달리기, 사이클)
- 수치 입력창: 운동 시간(분), 걸음 수, 거리(km)
- 슬라이더: 오늘 운동 강도 선택(약함~강함)
- 라디오 버튼 / 선택창: 운동 유형 선택(유산소, 근력, 스트레칭)
- 버튼: "기록 저장", "초기화", "결과 보기"
- 텍스트 출력 영역: 오늘의 운동 요약과 칼로리 계산 결과, 응원 메시지

## 4. 화면 스케치 (텍스트)
- [상단: 제목]
  - "오늘의 운동 기록"
  - "매일 운동을 쉽게 기록하고 확인해보세요"

- [중단: 입력 영역]
  - 운동 종류 입력창
  - 운동 시간 입력창
  - 걸음 수 / 거리 입력창
  - 운동 강도 슬라이더
  - 운동 유형 선택 라디오 버튼

- [중단 하단: 동작 버튼]
  - "기록 저장" 버튼
  - "초기화" 버튼

- [하단: 결과 메시지]
  - "오늘 운동 요약"
  - 입력한 운동 정보와 예측 칼로리 소모
  - "잘했어요! 내일도 함께해요" 같은 동기 부여 문구
